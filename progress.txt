## Codebase Patterns

- Backend group models: Contact (group_type enum), Conversation (group_type enum, column renamed from conversation_type via migration 20260227135739), ConversationGroupMember (role enum: member/admin, is_active for soft delete)
- Migrations: use `execute("ALTER INDEX IF EXISTS old_name RENAME TO new_name")` instead of `rename_index` to avoid PG DDL transaction issues
- Baileys provider: all group operations are in WhatsappBaileysService; new group endpoints must be defined as abstract methods on the base service first
- Frontend routing uses conversationType (mention/participating/unattended) — never confuse with the model field group_type
- Avatar placeholder colors: AVATAR_COLORS palette, index = name.length % 6 (6 pairs light + dark)
- Mention format stored in messages: [@Name](mention://user/ID/Name) or [@Name](mention://team/ID/Name) — group contacts use mention://contact/ID/Name
- ConversationBasicFilter dropdown uses SelectMenu from components-next/selectmenu; emits changeFilter and updates UI settings via updateUISettings
- Advanced filter attributes defined in advancedFilterItems/index.js + filterAttributeGroups; backend keys in lib/filters/filter_keys.yml
- All new Vue components must use <script setup> + Composition API; Tailwind only (no custom/scoped CSS)
- ActionCable events flow through a message handler that dispatches to Vuex; group_synced payload includes all member data — no re-fetch needed
- i18n: always add both en (app/javascript/dashboard/i18n/locale/en/) and pt_BR translations
- Editor.vue @ mention: `suggestionsPlugin` isAllowed gates dropdown trigger; TagAgents for private (agents/teams), TagGroupMembers for group non-private (contacts); both use `insertSpecialContent('mention', content)` with content.type determining mentionType in serialization
- Sender type comparison: Contact.push_event_data returns `type: 'contact'` (lowercase) but SENDER_TYPES.CONTACT is `'Contact'` (PascalCase) — always use case-insensitive comparison
- Tailwind spacing: `last:mt-N` only applies to :last-child; for consistent spacing between N children use `flex flex-col gap-N` on parent
- Group settings persisted in contact.additional_attributes: announce (bool), restrict (bool), joinApprovalMode (bool), memberAddMode, invite_code (string)
- GroupContactInfo.vue has `inboxPhone` computed that returns inbox phone_number; use for self-identification (You badge) and admin check
- Channel::Whatsapp delegates group methods to provider_service; new service methods need `delegate :method_name, to: :provider_service` in channel model
- Phone number normalization: inbox.phone_number may have `+` prefix while contact.phone_number may not — strip `+` before comparison
- Baileys API routes: check ROUTES_SUMMARY.md for correct endpoints; some routes differ from initial assumptions (e.g. group-request-participants-list, not group-join-requests)
